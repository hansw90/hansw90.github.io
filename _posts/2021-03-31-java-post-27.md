### 2. Group ID

groupId 에는 프로젝트마다 구별할 수 있는 고유한 이름이다. 보통은 java 패키지 네이밍을 일컫는다.

ex) org.apache.maven, org.apache.commons 

이 규칙은 강제적인것은 아니지만

groupId에 많은 하위 group을 만들때, 즉 멀티 프로젝트를 생성할때, 새로운 식별자만 부모 group id 뒤에 붙이는 식으로 사용하여 관리할 수 있다.
